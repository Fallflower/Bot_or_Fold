# Bot or Fold

> *Texas Hold'em in Your Terminal — 支持标准局和短牌的 CLI 德州扑克*

一个完整的德州扑克游戏，支持标准模式（52张）和短牌模式（36张），单人对抗电脑，拥有 CLI 交互界面、ASCII 艺术牌面、牌型评估、胜率计算等功能。

## 游戏模式

程序启动时可以选择游戏模式：

```
=== Select Game Mode =====
=== 1. Standard    [1] ===
=== 2. Short Deck  [2] ===
=== QUIT         [Esc] ===
==========================
```

| 模式 | 牌数 | 牌面 | 牌型规则 |
|------|------|------|----------|
| **Standard**（标准） | 52张（2~A） | 标准 | 同花 < 葫芦 |
| **Short Deck**（短牌） | 36张（6~A） | 去掉 2~5 | **同花 > 葫芦** |

短牌模式移除了点数 2~5 的牌，并调整了牌型大小顺序（同花概率更低因此高于葫芦）。

## 快速开始

### 环境要求
- C++17 编译器（g++ / MinGW）
- GNU Make

### 构建

```bash
make
```

### 运行

```bash
# 一键启动（默认参数：Standard 9人 200筹码 人类在Button）
make run

# 或直接运行并传入参数
./main.exe <玩家名> <人数> <起始筹码> <人类玩家位置> <模式>

# 示例：6人短牌局，1000 BB筹码，人类在庄位
./main.exe "MyName" 6 1000 0 2
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 玩家名 | 你的昵称 | 交互式输入 |
| 人数 | 2~15+ 人 | 交互式输入 |
| 起始筹码 | 每人起始筹码数 | 交互式输入 |
| 人类玩家位置 | 座位索引（0 ~ 人数-1） | 交互式输入 |
| 模式 | 1=标准, 2=短牌 | 交互式输入 |

省略参数时程序会依次提示输入；传入部分参数则跳过对应提示。

## 游戏操作

游戏过程中，轮到你行动时会显示菜单：

**情况一：无人下注（可过牌）**
```
=== Check          [1] ===
=== Bet            [2] ===
=== Allin          [3] ===
=== QUIT         [Esc] ===
```

**情况二：有人下注（筹码充足）**
```
=== Fold           [1] ===
=== Call      N BB [2] ===
=== Raise          [3] ===
=== Allin          [4] ===
=== QUIT         [Esc] ===
```

**情况三：筹码不足（全下或弃牌）**
```
=== Fold           [1] ===
=== Allin to Call  [2] ===
=== QUIT         [Esc] ===
```

- 按 `Esc` 键直接退出游戏
- Bet/Raise 时按提示输入下注额

### 短牌牌型调整

短牌中移除了 2~5 点数，并调整了牌型大小顺序：

| 标准牌型顺序 | 短牌牌型顺序 |
|:---:|:---:|
| ... | ... |
| 同花 (5) | **葫芦** (5) |
| 葫芦 (6) | **同花** (6) |
| ... | ... |

由于短牌中少了 2~5 点数，同花出现概率低于葫芦，因此同花 > 葫芦。

评估算法：
- **基础评估**（`HandType::evaluate`）：支持 2~7 张牌的规则评估，处理顺子、同花、A2345（标准）/ A6789（短牌）等特殊牌型
- **查表评估**（`advancedEvaluate`）：预计算 + 位运算哈希，快速评估 C(7,2)=21 种组合取最佳

### 核心类

| 类 | 职责 |
|------|----------|
| `Card<NumT>` | 单张牌模板类，支持 `toString` / `toColorString` / `toAsciiArt` 多种输出 |
| `Deck<NumT>` | 牌堆模板，提供洗牌、发牌、剩余牌计算，支持标准/短牌 |
| `Player` | 抽象基类，定义 `makeAction()` 纯虚接口 |
| `HumanPlayer` | 通过 CLI 控制台菜单获取玩家决策 |
| `BotPlayer` | 基于 equity vs potOdds 的决策 AI，支持价值下注、半诈唬和随机诈唬 |
| `HandType<NumT>` | 基础规则评估模板，支持标准/短牌不同牌型顺序 |
| `Position` | 根据人数自动分配座位（支持 2~15+ 人） |
| `Game<NumT>` | 主控模板类：状态流转、下注管理、底池计算、胜率模拟、赢家判定 |

### 游戏状态机

```
Pre-flop (0) → Flop (1) → Turn (2) → River (3) → End (4)
```

- 每轮所有玩家 Check 或跟注后自动进入下一阶段
- 仅剩 1 人未弃牌时游戏提前结束
- All-in 玩家自动跳过（`atag` 标记），不参与行动轮次

### 电脑玩家策略 (待优化)

BotPlayer 基于自身胜率（equity）与 pot odds 比较做决策：

- **无人下注**: equity > 60% 时主动价值下注(70%底池)，否则 8% 频率诈唬开池，其余过牌
- **有利可图 (equity > potOdds + 5%)**: 主要跟注，15% 频率半诈唬加注(3x)
- **强牌 (equity > 60%)**: 80% 加注(3x)，20% 跟注
- **不利局面 (equity < potOdds)**: 弃牌为主，10% 频率随机诈唬加注(4x)
- **筹码不足**: 无法跟注时直接弃牌或全下

### 胜率计算

基于蒙特卡洛模拟，分两种场景：

**calcEquity（玩家视角）** — 仅知自身手牌 + 已发公共牌，随机补全对手手牌后模拟。默认 12288 次，用于 BotPlayer 决策时评估自身 equity。

**calcWinRate（上帝视角）** — 已知所有玩家手牌 + 公共牌，从剩余牌堆随机补全未发公共牌后模拟。使用多线程加速（默认 20000 次 × CPU 核心数）。河牌发出后无需模拟，直接判定赢家。

两种方法均按平分底池计算胜率（多人平局时每人获得 `1.0 / 获胜人数` 的权重）。

### 模板架构

核心类使用 `NumT` 模板参数区分标准牌和短牌：

```cpp
enum class CARDNUM { NUM_2, NUM_3, ..., ACE };          // 13 ranks
enum class SHORT_CARDNUM { NUM_6 = 4, NUM_7, ..., ACE }; // 9 ranks

// 使用示例
Game<CARDNUM>       standardGame;       // 标准德州扑克
Game<SHORT_CARDNUM> shortDeckGame;      // 短牌
```

牌型数据通过两个独立的查表文件初始化：
- `ranker_data.h` / `ranker_data.cpp` — 标准牌（52张）
- `ranker_data_short.h` / `ranker_data_short.cpp` — 短牌（36张）

## 开发

```bash
# 构建
make

# 清理
make clean

# 构建并运行（9人标准局，200筹码，人类在小盲）
make run

# 5个参数：玩家名/玩家数/初始筹码/玩家位置(0为小盲，(人数-1)为庄位)/游戏模式(1为标准，2为短牌)
# 任何未指定的参数都需要在程序的指导下输入
# e.g. 运行6人短牌
./main.exe "MyName" 6 200 0 2
```

所有中间文件输出到 `build/` 目录。
