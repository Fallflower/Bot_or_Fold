# Bot or Fold
> *Texas Hold'em in Your Terminal — 单人对抗 Bot 的 CLI 德州扑克*

一个完整的德州扑克游戏，支持单人与电脑对战，拥有 CLI 交互界面、牌型评估、胜率计算等功能。

## 快速开始

### 环境要求
- C++17 编译器（g++ / MinGW）
- GNU Make
- 资源文件：`resources/card5_dic_zipped.bin`（牌型评估查表文件）

### 构建

```bash
make
```

### 运行

```bash
# 一键启动（默认参数：Demo 9人 200筹码 人类在Button）
make run

# 或直接运行并传入参数
./main.exe <玩家名> <人数> <起始筹码> <人类玩家位置>

# 示例：6人局，1000 BB筹码，人类在庄位
./main.exe "MyName" 6 1000 0
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 玩家名 | 你的昵称 | 交互式输入 |
| 人数 | 2~9+ 人 | 交互式输入 |
| 起始筹码 | 每人起始筹码数 | 交互式输入 |
| 人类玩家位置 | 你在座位列表中的索引（0 ~ 人数-1） | 交互式输入 |

如果省略所有参数，程序会依次提示输入；如果传入部分参数则跳过对应提示。

## 游戏操作

游戏过程中，轮到你行动时会显示菜单：

**情况一：无人下注（可过牌）**
```
=== Check          [1] ===
=== Bet            [2] ===
=== Allin          [3] ===
=== QUIT         [Esc] ===
```

**情况二：有人下注（筹码充足）**
```
=== Fold           [1] ===
=== Call      N BB [2] ===
=== Raise          [3] ===
=== Allin          [4] ===
=== QUIT         [Esc] ===
```

**情况三：筹码不足（全下或弃牌）**
```
=== Fold           [1] ===
=== Allin to Call  [2] ===
=== QUIT         [Esc] ===
```

- 按 `Esc` 键直接退出游戏
- Bet/Raise 时按提示输入下注额

## 牌型评估

支持 **中英文双语牌型显示**：

| 牌型 | 显示示例 |
|------|----------|
| 高牌 | `高牌A` |
| 一对 | `对Q T踢`（QQ带T踢脚） |
| 两对 | `对Q对7 T踢` |
| 三条 | `三条Q T踢` |
| 顺子 | `顺子T` |
| 同花 | `同花Q` |
| 葫芦 | `Q 7的葫芦` |
| 四条 | `四条Q T踢` |
| 同花顺 | `同花顺T` |
| 皇家同花顺 | `皇家同花顺` |

使用 **查表算法（预计算 + 位运算哈希）** 快速评估 C(7,5)=21 种 5 张牌组合，取最佳牌型。

### 核心类

| 类 | 职责 |
|------|----------|
| `Card` | 单张牌（花色·点数），支持字符串/彩色/整数/哈希多种输出 |
| `Deck` | 52 张牌堆，提供洗牌、发牌、剩余牌计算 |
| `Player` | 抽象基类，定义 `makeAction()` 纯虚接口 |
| `HumanPlayer` | 通过 CLI 控制台菜单获取玩家决策 |
| `BotPlayer` | 基于 equity vs potOdds 的决策 AI，支持价值下注、半诈唬和随机诈唬 |
| `HandType` | 基础规则 + 预计算哈希表两种牌型评估 |
| `Position` | 根据人数自动分配座位（支持 2~15+ 人） |
| `Game` | 主控类：状态流转、下注管理、底池计算、胜率模拟、赢家判定 |

### 游戏状态机

```
Pre-flop (0) → Flop (1) → Turn (2) → River (3) → End (4)
```

- 每轮所有玩家 Check 或跟注后自动进入下一阶段
- 仅剩 1 人未弃牌时游戏提前结束
- All-in 玩家自动跳过（`atag` 标记），不参与行动轮次

### 电脑玩家策略

BotPlayer 基于自身胜率（equity）与 pot odds 比较做决策：

- **无人下注**: equity > 60% 时主动价值下注(70%底池)，否则 8% 频率诈唬开池，其余过牌
- **有利可图 (equity > potOdds + 5%)**: 主要跟注，15% 频率半诈唬加注(3x)
- **强牌 (equity > 60%)**: 80% 加注(3x)，20% 跟注
- **不利局面 (equity < potOdds)**: 弃牌为主，10% 频率随机诈唬加注(4x)
- **筹码不足**: 无法跟注时直接弃牌或全下

### 胜率计算

基于蒙特卡洛模拟，分两种场景：

**calcEquity（玩家视角）** — 仅知自身手牌 + 已发公共牌，随机补全对手手牌后模拟。默认 12288 次，用于 BotPlayer 决策时评估自身 equity。

**calcWinRate（上帝视角）** — 已知所有玩家手牌 + 公共牌，从剩余牌堆随机补全未发公共牌后模拟。使用多线程加速（默认 20000 次 × CPU 核心数）。河牌发出后无需模拟，直接判定赢家。

两种方法均按平分底池计算胜率（多人平局时每人获得 `1.0 / 获胜人数` 的权重）。
